1. Propósito único de la extensión

FakeNews Detector es una herramienta de apoyo a la decisión que verifica afirmaciones en el contenido de páginas web elegido por el usuario mediante modelos de lenguaje (LLM) de su propia elección. No es un detector infalible de mentiras; los resultados del LLM pueden contener errores, sesgos o información desactualizada. Verifica siempre de forma independiente las afirmaciones que sean críticas.

2. Qué datos se procesan y hacia dónde van

La extensión nunca envía datos a ningún servidor operado por el desarrollador. Todo el procesamiento se produce directamente entre tu navegador y los servicios que tú mismo configuras.

DatoDestinoCondición
Texto visible de la página que analizas API del proveedor de IA que tú seleccionas, usando tu propia clave Solo cuando inicias un análisis manual o activas el modo automático / silencioso
Audio de la pestaña capturada API del proveedor STT (transcripción) que tú configuras, usando tu propia clave STT Solo cuando inicias un análisis de audio manualmente
Audio del micrófono (modo vigilante) Mismo proveedor STT configurado por ti Solo tras conceder el permiso de micrófono explícitamente desde la página de Configuración
Título de la página, URL, etiqueta de veredicto, puntuación de confianza y primer aviso de riesgo (hasta 180 caracteres) api.telegram.org — tu propio bot Solo si configuras un token de bot y un chat ID en Configuración, y solo para veredictos de riesgo (falsa / dudosa) en modo silencioso
JSON de alerta (origen, email de destino, título, URL, veredicto, etiqueta de veredicto, confianza, primera afirmación marcada, marca de tiempo). El email de destino se incluye para que la automatización receptora sepa a dónde reenviar la alerta. URL de webhook que tú introduces Solo si configuras una URL de webhook válida en Configuración; el receptor puede ser n8n, Zapier, Make o cualquier endpoint propio
Enlace de donación buymeacoffee.com Solo si haces clic en el chip de donación; no se envían datos personales
El desarrollador no recibe, almacena ni ve ninguno de los datos anteriores. No existe ningún servidor de retransmisión.

3. Datos almacenados localmente

Los siguientes datos se guardan únicamente en tu dispositivo:

  • chrome.storage.local (permanente hasta desinstalar): clave de API, proveedor seleccionado, modelo, URL base, clave STT, URL base STT, modelo STT, intervalo de análisis, tema visual, modo automático, modo silencioso, token del bot de Telegram, chat ID de Telegram, URL del webhook y email de destino.
  • chrome.storage.session (se borra al cerrar el navegador): estado de análisis por pestaña, historial de sesión de la pestaña activa (hasta 50 entradas), mapas de deduplicación de URLs y contador de alertas del modo silencioso.

Ninguno de estos datos abandona tu dispositivo salvo los envíos directos descritos en la tabla anterior.

4. Terceros — proveedores seleccionados por el usuario

Según el proveedor que elijas, tu navegador se conectará directamente a uno de los siguientes endpoints. Cada uno se rige por su propia política de privacidad:

  • Anthropic — api.anthropic.com
  • OpenAI — api.openai.com
  • Google Gemini — generativelanguage.googleapis.com
  • DeepSeek — api.deepseek.com
  • Alibaba / Qwen — dashscope-intl.aliyuncs.com
  • Moonshot / Kimi — api.moonshot.ai
  • Zhipu / GLM — api.z.ai
  • MiniMax — api.minimax.io, api.minimaxi.com
  • xAI / Grok — api.x.ai
  • Mistral — api.mistral.ai
  • Groq — api.groq.com
  • OpenRouter — openrouter.ai
  • Ollama (local) — localhost / 127.0.0.1 — ningún dato sale de tu máquina
  • Telegram (alertas opcionales) — api.telegram.org

La extensión también admite endpoints personalizados (Personalizado) y servidores Whisper autoalojados; en ese caso el tráfico va al servidor que tú mismo controlas.

5. Permisos explicados

PermisoPor qué es necesario
storage Guardar tu clave de API, configuración y estado de sesión localmente en tu dispositivo.
tabs Leer la URL y el título de la pestaña activa para el análisis automático/silencioso y para incluir esa información en las alertas de Telegram o webhook.
activeTab Acceder al contenido de texto de la pestaña que el usuario elige analizar (botón manual «Analizar texto» o análisis automático en segundo plano).
tabCapture Capturar el stream de audio de la pestaña activa para el análisis de audio/vídeo (YouTube, sitios de streaming, etc.). Solo disponible en Chrome y Edge.

Acceso a hosts <all_urls> Inyectar el script de contenido en cualquier página que el usuario decida analizar. Es necesario porque el usuario puede navegar a cualquier sitio web y elegir verificarlo. También lo requieren los modos automático y silencioso, que el usuario activa explícitamente para analizar páginas en segundo plano.
Acceso al micrófono Solo en el modo vigilante. El permiso lo concede el usuario explícitamente a través del diálogo del navegador desde la página de Configuración. El audio va directamente al proveedor STT configurado, sin pasar por ningún servidor del desarrollador.

6. Lo que la extensión NO hace

  • No recopila analíticas, métricas de uso ni historial de navegación.
  • No muestra anuncios ni comparte datos con redes publicitarias.
  • No vende ni alquila ningún dato.
  • No utiliza los datos para fines distintos del análisis de contenido que el usuario inicia.
  • No retiene ningún dato en servidores propios porque no existen servidores propios.

7. Control del usuario

  • Toda funcionalidad es opt-in. El análisis manual requiere un clic; el modo automático y el modo silencioso requieren activarlos expresamente.
  • El botón Reiniciar del panel borra los resultados y el historial de la sesión activa.
  • Desinstalar la extensión elimina todos los datos locales (claves, configuración, historial).
  • Puedes revocar el permiso de micrófono en cualquier momento desde la configuración del navegador.

8. Cambios en esta política

Si se realizan cambios materiales, se actualizará la fecha de vigencia en esta página y se reflejará en el historial del repositorio de GitHub: github.com/tonirojas/fakenews-detector. Recomendamos revisar esta política periódicamente.

1. Single purpose

FakeNews Detector is a decision-support tool that fact-checks claims in web page content chosen by the user, using a large-language model (LLM) provider and API key selected and supplied by the user. It is not an infallible lie detector. LLM outputs may contain errors, biases, or outdated information. Always verify critical claims independently.

2. What data is processed and where it goes

The extension never transmits data to any server operated by the developer. All processing happens directly between your browser and the services you configure.

DataDestinationWhen
Visible text of the page you analyze The AI provider API you select, using your own API key Only when you trigger manual analysis or enable auto / silent mode
Captured tab audio The STT (transcription) provider API you configure, using your own STT key Only when you manually start an audio analysis
Microphone audio ("vigilante" mode) Same STT provider configured by you Only after you explicitly grant microphone permission from the Settings page
Page title, URL, verdict label, confidence score, and first flagged claim (up to 180 chars) api.telegram.org — your own bot Only if you configure a bot token and chat ID in Settings, and only for risky verdicts (false / uncertain) in silent mode
Alert JSON (source, destination email, title, URL, verdict, verdict label, confidence, first flagged claim, timestamp). The destination email is included so the receiving automation knows where to forward the alert. Webhook URL you provide Only if you configure a valid webhook URL in Settings; the receiver may be n8n, Zapier, Make, or any custom endpoint you control
Donation link buymeacoffee.com Only if you click the donation chip; no personal data is transmitted
The developer does not receive, store, or view any of the above data. There is no relay server.

3. Data stored locally on your device

The following data is stored only on your device:

  • chrome.storage.local (persists until uninstall): API key, selected provider, model, base URL, STT key, STT base URL, STT model, analysis interval, visual theme, auto mode flag, silent mode flag, Telegram bot token, Telegram chat ID, webhook URL, and destination email.
  • chrome.storage.session (cleared when the browser closes): per-tab analysis state, active tab session history (capped at 50 entries), URL dedup maps (10-minute auto-mode and 30-minute alert-mode windows), and silent-mode alert badge counter.

None of this data leaves your device except through the direct transmissions described in the table above.

4. Third parties — user-selected providers

Depending on the provider you choose, your browser will connect directly to one of the following endpoints. Each is governed by its own privacy policy:

  • Anthropic — api.anthropic.com
  • OpenAI — api.openai.com
  • Google Gemini — generativelanguage.googleapis.com
  • DeepSeek — api.deepseek.com
  • Alibaba / Qwen — dashscope-intl.aliyuncs.com
  • Moonshot / Kimi — api.moonshot.ai
  • Zhipu / GLM — api.z.ai
  • MiniMax — api.minimax.io, api.minimaxi.com
  • xAI / Grok — api.x.ai
  • Mistral — api.mistral.ai
  • Groq — api.groq.com
  • OpenRouter — openrouter.ai
  • Ollama (local) — localhost / 127.0.0.1 — no data leaves your machine
  • Telegram (optional alerts) — api.telegram.org

The extension also supports custom endpoints and self-hosted Whisper servers; in that case traffic goes to the server you control.

5. Permissions — plain-language explanation

PermissionWhy it is needed
storage Store your API key, settings, and session state locally on your device.
tabs Read the URL and title of the active tab for automatic/silent-mode analysis and to populate Telegram or webhook alert messages with page context.
activeTab Access the text content of the page the user explicitly chooses to analyze (manual "Analizar texto" button, or background analysis in auto/silent mode).
tabCapture Capture the audio stream of the active tab for audio/video fact-checking (YouTube, streaming sites, etc.). Chrome and Edge only; automatically disabled in Firefox.
offscreen Create an offscreen document that processes tab audio and microphone input using MediaRecorder and getUserMedia. Required by the MV3 architecture to use these APIs outside the service worker.
sidePanel Show analysis results, session history, and the conclusion view in Chrome's built-in side panel.

Microphone access Vigilante mode only. Permission is granted explicitly by the user via the browser's permission dialog from the Settings page. Audio goes directly to the configured STT provider — not through any developer server.

6. What the extension does NOT do

  • Does not collect analytics, usage metrics, or browsing history.
  • Does not display ads or share data with advertising networks.
  • Does not sell or rent any data.
  • Does not use data for any purpose other than the content analysis initiated by the user.
  • Does not retain any data on developer-operated servers because no such servers exist.

7. User controls

  • All functionality is opt-in. Manual analysis requires a click; auto mode and silent mode must be explicitly enabled.
  • The Reiniciar (Reset) button in the panel clears results and session history for the active tab.
  • Uninstalling the extension removes all local data (keys, settings, history).
  • You can revoke microphone permission at any time from the browser settings.

8. Changes to this policy

If material changes are made, the effective date on this page will be updated and the change will be reflected in the GitHub repository history: github.com/tonirojas/fakenews-detector. We recommend reviewing this policy periodically.
